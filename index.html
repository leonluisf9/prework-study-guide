Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines title for the webpage. It is important for SEO.
  • The style element defines css styles associated with a website.
  • The base element defines base url for a webpage.
  • The link element connects an external resource to an HTML document.
  • The meta element defines metadata such as character set, description, keywords, author and viewport.
  • The paragraph element represents a block.
  • The unordered list element.
  • The ordered list element.
  • The list item element.
  • The image element has a src attribute that defines image location.
  • The linebreak element creates a line of empty space.
  • The anchor element creates links to same or other webpages. It has href attribute points to url for the link

CSS

  • Used to define how webpage content is presented to users.
  • Defines colors, aesthetics, visual layout and organization.
  • 3 ways to create css file: Inline css, Internal and External css style sheet.
  • External css style sheet is most common because allow developers to keep all of their css rules in a separate file, maek changes easier.
  • External css style sheet requires use of link element placed in HTML head.
  • In a CSS rule a selector defines elements or attributes to which rules and declarations will apply.
  • In a CSS rule declaration contains 2 components css property and value.
  • A padding indicates how much space we want around the content inside an element.
  • A margin indicates how much space we want around the outside of an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switch to the main branch
  • git checkout -b feature/myaction: creates a new feature branch named feature/myaction
  • code: opens visual studio code on the current directory
  • pwd: displays working directory
  • git add -A: add all changes in the working directory to staging area
  • git commit -m "description": take all changes made locally and push them to local repository with a description of problem commits solves
  • git clone: copies entire Git repository to work on a copy without impact to original one
  • git pull origin: pull changes from origin and merge them to local checked-out branch
  • git push: share local commits and push them to a remote repository
